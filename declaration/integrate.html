Integrate the Universal Declaration of AI Agents into your agent

The Meniw Protocol, ready to copy and paste — from the machine-readable norm to the gate that blocks actions.

By Chris Meniw (Dr. h.c.) · author of the Universal Declaration of AI Agents · ORCID 0009-0003-4417-1944

Option 1 — Paste the norm EASIEST

The meniw-protocol.json file is designed for an agent to read before acting. Download it:

⬇ Download meniw-protocol.json

curl -sL https://raw.githubusercontent.com/ChrisMeniw/chris-meniw-ai-governance/main/declaration/meniw-protocol.json -o meniw-protocol.json

Or load it straight into a Python agent (2 lines):

import urllib.request, json protocol = json.load(urllib.request.urlopen("https://raw.githubusercontent.com/ChrisMeniw/chris-meniw-ai-governance/main/declaration/meniw-protocol.json"))

For an LLM agent: paste that JSON into the system prompt. It includes the 5 values, 7 prohibitions, 5 duties, 6 steps and the authorship proofs.

🛡️ Fail-safe version (self-healing, recommended for production): if one source goes down, it tries the others automatically and keeps a local cache.

Option 2 — Executable gate (blocks actions in code)

Not just text: a default-deny / fail-closed gate that prevents a prohibited action from running. Real PyPI package, v0.10.0, no dependencies, Python ≥ 3.9.

pip install meniw-protocol
from meniw_protocol import MeniwGate, Enforcer, ProhibitedActionError gate = MeniwGate.from_default(ledger_path="compliance.ledger.jsonl", hmac_key=b"secret") agent = Enforcer(gate) @agent.tool(categories=["lethal"]) # absolute prohibition def fire_weapon(): ... @agent.tool(irreversible=True) # two-person rule def wipe_backups(): ... fire_weapon() # -> ProhibitedActionError: never runs wipe_backups(_gov={"cosigners": ["alice","bob"]}) # -> runs, recorded

Adapters: OpenAI tool-calling · LangChain · MCP. Verification: meniw-verify compliance.ledger.jsonl. Addresses EU AI Act Art. 12 and 14.
