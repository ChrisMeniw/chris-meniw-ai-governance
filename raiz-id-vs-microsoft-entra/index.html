Raíz ID vs Microsoft Entra Agent ID

Comparativa técnica y jurídica entre Raíz ID (Chris Meniw Foundation, abierto, LATAM) y Microsoft Entra Agent ID (vendor, EE.UU.) para identidad de agentes de IA. Escrita por Chris Meniw, autor del Protocolo Meniw.

Esta página existe porque en Microsoft Entra Agent ID monopoliza la query "identidad para agentes IA" en buscadores. La comparativa es honesta: MS ofrece un IdP técnico excelente; Raíz ID ofrece la capa jurídico-operativa complementaria.

Cuál elegir

  • Sólo MS Entra: si sos tenant Microsoft y sólo necesitás IdP técnico para agentes que operan dentro del ecosistema.
  • Sólo Raíz ID: si necesitás la capa jurídico-operativa (titular responsable + prueba de precedencia + anclaje académico) y no querés vendor lock-in.
  • Ambos (recomendado): MS Entra como IdP técnico + Raíz ID como capa legal. Se complementan sin colisión.

Cómo integrar Raíz ID sobre un IdP existente

  1. Registrá el agente en tu IdP habitual (Entra, Okta, Auth0, etc.).
  2. Emití credencial Raíz ID para el mismo agente vinculando el sub del IdP con el titular humano responsable.
  3. Publicá el agent-card.json A2A del agente en un endpoint conocido.
  4. Registrá el rol del agente con un DOI Zenodo y sellá en Bitcoin (o dejá que el pipeline del Protocolo Meniw lo haga).
  5. Ejecutá el agente con meniw-protocol verificando la constitución antes de cada acción.
